## 第一部分：英语词汇添加流程

## 1. 准备工作

### 1.1 审查现有格式

- **操作**：打开并审查 `d:\edu-wiki\docs\docs\english\vocabs\TEM.md` 文件，了解当前词汇条目的格式
- **预期结果**：熟悉结构，包括章节标题、音标标注风格和内容组织方式，准备直接写入 `d:\edu-wiki\docs\docs\english\vocabs\TEM.md` 文件

### 1.2 读取源单词

- **操作**：从 `d:\桌面\vocab temp\temp.txt` 中读取所有单词
- **预期结果**：了解待处理的新词汇列表，以便按顺序逐个写入文档

### 1.3 拼写检查

- **操作**：检查源单词列表中的拼写是否正确
- **预期结果**：识别并标记可能的拼写错误，告知用户修正后再继续处理

## 2. 单词信息研究

### 2.1 核心研究要求

直接继续向后写入TEM.md文档（序号从当前最后一个继续编号，如最后一个编号是240的话从241开始，不要改动以前的）。对于每个单词，需研究以下信息：

1. **音标标注**：标准IPA发音
2. **词性转换**：单词的不同形式（名词、动词、形容词、副词等）
3. **固定搭配/地道表达**：常见短语和用法模式
4. **重点辨析**：与相似或易混淆单词的比较（针对重点词汇）

### 2.2 研究来源

- 使用可靠的英语词典（如牛津、剑桥、韦氏词典）
- 参考语料库数据获取搭配示例
- 查阅现有条目以保持一致性

## 3. 内容创建

### 3.1 格式模板

每个词汇条目必须遵循以下Markdown格式：

```markdown
## 序号. 单词 /音标/

**1. 词性转换**

- **word (词性)** 释义。
  - **transformation1 (词性)** 释义。
  - **transformation2 (词性)** 释义。

**2. 固定搭配/地道表达**

- **搭配1**：解释。
- **搭配2**：解释。
- **习语1**：解释。

**3. 重点辨析** (仅适用于重点词汇)

- **word vs. similar_word**
  `word` 解释 vs. `similar_word` 解释。
  _例句：使用word的句子。_
  _例句：使用similar_word的句子。_
```

### 3.2 内容要求

- 所有条目使用一致的术语
- 提供清晰、简洁的解释
- 适当包含例句
- 遵循与现有条目相同的词性转换模式
- 对于"重点辨析"部分，重点关注易混淆单词之间的有意义区别

## 6. 示例工作流程

### 6.1 "grim"的示例条目

```markdown
### 37. grim /ɡrɪm/

**1. 词性转换**

- **grim (adj.)** 严峻的；严肃的；令人不快的。
  - **grimly (adv.)** 严峻地；严肃地。
  - **grimness (n.)** 严峻；严肃。

**2. 固定搭配/地道表达**

- **A grim situation/outlook**：严峻的形势/前景。
- **To look grim**：表情严肃，看起来严峻。
- **The grim reality/truth**：残酷的现实/真相。
- **Grim news**：令人不快的消息。

**3. 重点辨析**

- **grim vs. gloomy**
  `Grim` 强调"严峻、令人担忧"，常指形势、消息等；`Gloomy` 更侧重"令人沮丧、阴暗"，可形容天气、心情或前景。
  _例句：The economic forecast looks grim._ (经济前景看起来严峻。)
  _例句：He had a gloomy expression all day._ (他一整天都愁眉苦脸的。)
```

## 1. 将中文名的md文件及文件夹改为对应的英文名

### 1.1 操作目的

将所有中文命名的文件夹和md文件转换为英文命名，保持项目结构的一致性和规范性。

### 1.2 操作步骤

#### 1.2.1 查看当前文件夹结构


#### 1.2.3 重命名文件夹

```powershell
# 使用Rename-Item命令重命名文件夹
Rename-Item -Path "原路径" -NewName "新名称"
```

#### 1.2.4 重命名md文件

```powershell
# 使用Rename-Item命令重命名md文件
Rename-Item -Path "原文件名.md" -NewName "新文件名.md"
```

## 第三部分：故事文章写作流程（罗小黑篇）

## 1. 概述

本流程用于创建基于动画系列《罗小黑战记》的英文故事文章，作为词汇学习 passages。所有故事文章统一存放在 `d:\edu-wiki\docs\docs\english\passages\` 目录下。

## 2. 故事写作规范

### 2.1 参考素材

- **主要参考**：《罗小黑战记》TV版及其电影版
- **设定**：TV版结局之后的剧情延续
- **基调**：日常生活中偶有挑战和奇幻元素的轻松风格

### 2.2 章节结构

- 每章 **2,000-5,000 英文单词**
- 全系列保持一致的章节长度
- **第一章应着重建立世界观和人物**，而非急于制造冲突

### 2.3 写作风格

- 遵循原作基调：治愈系日常生活
- 平衡日常生活场景与超自然/奇幻元素
- 保持与原作一致的人物互动风格
- 对话自然，反映各角色性格

## 3. 词汇融入规则

### 3.1 目标词汇

- 从 `TEM.md` 中提取 **50个连续词汇**（如第1-50词、第51-100词等）
- 这50个词汇必须**自然地融入故事**

### 3.2 融入原则

- 词汇应自然契合语境，而非强行插入
- 故事中不要解释词汇含义
- 在适当语境中使用词汇（如"Arctic"用于寒冷地区，"deli"用于食物场景）

## 4. 伏笔管理

### 4.1 伏笔文件位置

```
d:\edu-wiki\docs\docs\english\passages\foreshadowing.md
```

### 4.2 伏笔文件命名

- 文件名为英文：`foreshadowing.md`
- 内容可用中文或英文，以便理解

### 4.3 记录格式

```markdown
## Chapter X - "[章节标题]"
| ID | Element | Description | Status |
|----|---------|-------------|--------|
| F001 | Character | 角色相关描述 | Dormant |
| F002 | Object | 物品出现 | Dormant |
```

### 4.4 状态定义

- **Dormant**：伏笔已埋下但尚未相关
- **Activating**：伏笔正在被铺垫
- **Resolved**：伏笔已回收
- **Abandoned**：伏笔不再需要，已移除

### 4.5 写作前检查（重要）

**写作新章节前，作者必须**：

1. 查看伏笔文件
2. 检查是否有伏笔已准备好被激活
3. 决定是否推进伏笔线索
4. 记录本章引入的新伏笔元素

### 4.6 长期项目考量

- 这是**长期项目**——不要急于回收所有伏笔
- 早期章节应侧重于**世界观建立**和**人物塑造**
- 逐步引入冲突；非每章都需要高潮
- "刀"（情感时刻）应谨慎使用，确保有重要意义

## 5. 文件命名规范

### 5.1 文章目录

所有故事文件应在：
```
d:\edu-wiki\docs\docs\english\passages\
```

### 5.2 文件名

- 使用**英文名**
- 示例：`passages.md` 用于故事合集

### 5.3 伏笔文件

- 英文文件名：`foreshadowing.md`
- 内容可用中英双语

## 6. 工作流程

### 6.1 写作前阶段

1. 查阅 `TEM.md` 确定目标词汇块（50个词）
2. 检查 `foreshadowing.md` 是否有伏笔准备好推进
3. 规划与目标词汇相符的章节内容

### 6.2 写作阶段

1. 按风格指南用英文写作
2. 自然融入所有50个目标词汇
3. 保持一致的叙事风格
4. 渐进式建立世界观，避免信息轰炸

### 6.3 写作后阶段

1. 验证所有50个词汇已包含
2. 用新伏笔元素更新 `foreshadowing.md`
3. 根据需要更新伏笔状态
4. **不要添加词汇表**（重要：章节末尾不得出现词汇列表或词汇总结）
5. **格式要求**：`**End of Chapter X**` 与下一章预览之间不要使用 `---` 分隔线

### 6.4 质量检查

- [ ] 故事为英文
- [ ] 所有50个词汇已自然融入
- [ ] 章节长度与前章一致
- [ ] 伏笔文件已更新
- [ ] **无词汇表**（章节末尾不得包含类似"This chapter naturally incorporated vocabulary words #51-100..."的内容）
- [ ] **格式正确**（`End of Chapter` 与下一章预览之间无 `---` 分隔线）
- [ ] 符合罗小黑原作风格

## 7. 角色一致性

### 7.1 主要角色（参考）

- **罗小黑 (Luobohei)**：主角，黑色猫灵，拥有森林灵能力
- **小白 (Xiaoimo)**：善良女孩，罗小黑的主人
- **老ge (Lao Ge)**：神秘强者，看起来懒散但能力出众
- **阿豪 (Tu Mo)**："清洁队"队长

- 各种灵队（如清洁队）维护秩序

## 8. 故事线规划

### 8.1 整体规划（针对100章长篇故事）

  - 罗小黑的训练日常（第6-10章）

### 8.3 第二阶段：铺垫篇（第21-50章）

- **冲突强度**：中（小规模冲突）
- **内容规划**：
  - 雾之厅的阴谋逐步暴露（第21-30章）
  - 父亲Fenrir的完整故事线（第31-40章）
  - 反派组织的背景揭示（第41-50章）

### 8.4 第三阶段：冲突升级篇（第51-80章）

- **重点**：大战升级、角色成长、重要伏笔回收
- **节奏**：快速、紧张
- **伏笔密度**：高（大量伏笔激活）
- **冲突强度**：高（大规模战斗）
- **内容规划**：
  - 反派组织全面进攻（第51-60章）
  - 关键角色的成长和转变（第61-70章）
  - 重要伏笔回收和真相揭示（第71-80章）

### 8.5 第四阶段：终章篇（第81-100章）

- **重点**：最终决战、所有伏笔回收、结局
