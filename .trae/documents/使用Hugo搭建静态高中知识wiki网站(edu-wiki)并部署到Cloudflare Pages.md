# 使用MkDocs+Material主题搭建静态高中知识wiki网站(edu-wiki)并部署到Cloudflare Pages

## 关于ctf-wiki使用的主题
通过查看ctf-wiki的配置文件，我确认ctf-wiki使用的是**MkDocs**静态站点生成器，搭配**Material for MkDocs**主题。这是一个功能强大、外观现代化的主题，非常适合构建知识库和wiki类型的网站。

## 1. 项目初始化和配置

### 1.1 技术栈选择
- **MkDocs**：静态站点生成器，基于Python，简单易用
- **Material for MkDocs**：现代化的文档主题，功能丰富
- **Cloudflare Pages**：全球边缘部署，提供快速、可靠的静态网站托管

### 1.2 初始化项目
- 在当前目录(`d:\edu-wiki`)初始化MkDocs项目
- 安装必要的依赖：MkDocs和Material for MkDocs
- 配置基本的项目设置，包括标题、描述、主题等

## 2. 内容结构设计

### 2.1 目录结构
- 设计适合高中知识的目录结构，参考ctf-wiki的结构，例如：
  ```
  docs/
  ├── index.md          # 首页
  ├── 数学/              # 数学分类
  │   ├── 代数.md
  │   ├── 几何.md
  │   └── ...
  ├── 物理/              # 物理分类
  │   ├── 力学.md
  │   ├── 电磁学.md
  │   └── ...
  ├── 化学/              # 化学分类
  ├── 生物/              # 生物分类
  ├── 语文/              # 语文分类
  ├── 英语/              # 英语分类
  └── static/            # 静态资源目录
      ├── img/           # 图片资源
      ├── css/           # 自定义CSS
      └── js/            # 自定义JS
  mkdocs.yml              # MkDocs配置文件
  requirements.txt        # 依赖声明文件
  ```

### 2.2 主题和样式定制
- 使用Material for MkDocs主题
- 配置导航栏和侧边栏
- 定制主题颜色、字体等
- 启用搜索、标签页等功能
- 配置响应式设计，适配各种设备

## 3. 本地开发和测试

### 3.1 本地开发环境

### 3.2 内容创作

## 实施步骤

1. 安装Python和pip（如果尚未安装）

## 为什么选择MkDocs+Material主题

- **成熟稳定**：MkDocs是一个成熟的静态站点生成器，拥有活跃的社区支持
- **现代化外观**：Material主题提供了简洁、现代化的设计，适合知识型网站
- **功能丰富**：内置搜索、导航、响应式设计等功能
- **易于配置**：配置文件简洁明了，易于修改
- **Markdown支持**：完美支持Markdown语法，适合内容创作
- **扩展性强**：支持插件扩展，可以根据需要添加更多功能
- **快速构建**：构建过程快速，适合频繁更新的内容

这个计划将帮助您快速搭建一个功能完整、外观现代化的静态高中知识wiki网站，并成功部署到Cloudflare Pages。
